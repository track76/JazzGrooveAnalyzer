# Jazz Groove Analyzer (JGA)

## Official Method Specification

**Version 1.1 (Draft)**

**Author**

Angelo Tracanna

---

# 1. Introduction

The Jazz Groove Analyzer (JGA) is a computational framework designed for the analysis of ensemble rhythm in jazz performance.

Unlike conventional beat tracking systems, the JGA does not detect the beat directly.

Instead, it progressively reconstructs the ensemble metric reference by analysing the rhythmic interaction of the musical sources participating in the performance.

Within the JGA framework, the beat is therefore not considered an input but the final result of a hierarchical analytical process.

---

# 2. Philosophy

The JGA is based on one fundamental principle:

**The metric reference of an ensemble is not imposed. It emerges from the temporal cooperation of the musical sources contributing to the collective pulse.**

Consequently, the JGA does not search directly for the beat.

Instead, it reconstructs the musical process through which the ensemble generates its own metric organisation.

This philosophy fundamentally distinguishes the JGA from conventional beat tracking algorithms based exclusively on onset detection or statistical beat estimation.

---

# 3. Fundamental Principles

The behaviour of the JGA is governed by the following principles.

## Principle 1 — Acoustic Neutrality

Every detected acoustic event is initially considered a possible metric event.

No musical interpretation is performed during the detection stage.

---

## Principle 2 — Emergent Metric Relevance

Metric relevance is never assigned according to the instrument itself.

Instead, it emerges from the observed rhythmic behaviour of each musical source.

---

## Principle 3 — Periodic Behaviour

Only musical sources exhibiting stable periodic behaviour may contribute to the reconstruction of the ensemble metric reference.

---

## Principle 4 — Dynamic Metric Roles

Metric roles are dynamic.

A musical source may enter or leave the metric reference at any moment during the performance according to its musical behaviour.

---

## Principle 5 — Collective Metric Emergence

The Ensemble Metric Event (EME) is generated by the temporal consensus of the active Metric Sources.

No individual musician produces the EME.

It is an emergent property of the ensemble.

---

## Principle 6 — Metric Compatibility

Intentional rhythmic subdivisions do not interrupt the metric reference.

Triplets, duplets, quintuplets and every intentional metric subdivision are interpreted as legitimate transformations of the beat rather than timing deviations.

---

## Principle 7 — Human Timing

Natural human timing variability is considered an intrinsic characteristic of musical performance.

The JGA evaluates rhythmic coherence rather than mechanical temporal equality.

---

# 4. Analysis Pipeline

The complete analytical workflow of the JGA is organised as follows.

```text
Audio File

↓

Audio Normalization

↓

Source Separation

↓

AudioStem Generation

↓

Metric Source Recognition

↓

Pulse Detection

↓

Pulse Candidate Extraction

↓

Pulse Interval Construction

↓

Periodicity Analysis

↓

Persistence Analysis

↓

Ensemble Metric Event (EME)

↓

Metric Cluster Construction

↓

Ensemble Metric Reference

↓

Groove Analysis
```

Each analytical stage progressively increases the musical interpretation of the original audio signal.

---

# 5. Source Separation

The first analytical stage consists of separating the original recording into independent musical sources.

The current implementation employs **HTDemucs** as the source separation backend.

The separation backend is abstracted through the **BaseSeparator** interface, allowing future replacement without modifying the remaining analytical pipeline.

The output of this stage consists of a collection of **AudioStem** objects.

Each AudioStem represents one independent musical source that can be analysed separately.

Typical AudioStems include:

* Bass
* Drums
* Piano
* Guitar
* Vocals
* Other Instruments

---

# 6. Metric Source Recognition

The JGA does not assume that every musical source contributes equally to the ensemble pulse.

Each AudioStem is analysed independently in order to determine whether it actively participates in the construction of the ensemble metric reference.

The recognition process evaluates:

* rhythmic periodicity
* rhythmic persistence
* rhythmic continuity
* metric compatibility

Only sources presenting sufficiently stable rhythmic behaviour become **Metric Sources**.

Sources performing melodic, harmonic or improvisational functions remain available for expressive analysis but do not participate in the metric reconstruction.

Metric Sources may change dynamically throughout the performance.

---

# 7. Drum Pulse Carrier Recognition

The drum set represents a particular analytical case because multiple rhythmic components coexist within a single AudioStem.

The JGA therefore performs an additional analysis to identify which drum component currently carries the pulse.

Possible pulse carriers include:

* Ride Cymbal
* Hi-Hat
* Brushes
* Rim Shot
* Cross Stick
* Snare Drum
* Kick Drum

Only the active pulse carrier contributes to the reconstruction of the ensemble metric reference.

Whenever the drummer changes the pulse carrier during the performance, the JGA automatically updates the corresponding Metric Source.

---

# 8. Pulse Detection

Pulse Detection is performed independently for every recognised Metric Source.

Detected acoustic events become Pulse Candidates.

At this stage no musical interpretation is assigned.

Each Pulse Candidate remains a purely acoustic observation until validated by the subsequent analytical stages.

---

# 9. Pulse Interval Construction

Pulse Candidates are chronologically ordered.

Each pair of consecutive Pulse Candidates generates one Pulse Interval.

Pulse Intervals constitute the temporal basis for the estimation of rhythmic periodicity.

---

# 10. Periodicity Analysis

Periodicity measures the persistence of coherent rhythmic behaviour through time.

Unlike simple temporal regularity, periodicity allows natural human timing fluctuations while preserving the perception of a stable pulse.

The JGA therefore evaluates musical periodicity rather than mechanical precision.

---

# 11. Persistence Analysis

Persistence measures the temporal duration during which a musical source maintains coherent rhythmic behaviour.

Persistent rhythmic behaviour indicates stable participation in the ensemble metric organisation.

---

# 12. Ensemble Metric Event (EME)

The Ensemble Metric Event (EME) represents the elementary unit of metric reconstruction.

It is generated exclusively from the temporal consensus of the active Metric Sources.

Intentional metric subdivisions are interpreted as compatible transformations of the beat and therefore do not generate independent Ensemble Metric Events.

The EME represents the collective pulse perceived at the ensemble level rather than the timing of any individual performer.

---

# 13. Metric Cluster

Metric Clusters are constructed from the ordered succession of Ensemble Metric Events.

Unlike conventional beat tracking systems, Metric Clusters are not generated directly from detected transients.

Instead, they emerge from the collective metric behaviour reconstructed by the JGA.

Metric Clusters describe the dynamic evolution of the ensemble metric organisation.

---

# 14. Ensemble Metric Reference

The Ensemble Metric Reference represents the final pulse reconstructed by the Jazz Groove Analyzer.

It is progressively estimated from the temporal succession of Metric Clusters.

Consequently, the reconstructed metric reference corresponds to the emergent pulse of the ensemble rather than to the timing of any individual performer.

---

# 15. Confidence

Every analytical decision produced by the JGA is associated with a Confidence value.

Confidence quantifies the estimated reliability of:

* AudioStem separation
* Metric Source recognition
* Drum pulse carrier identification
* Ensemble Metric Events
* Metric Clusters
* Ensemble Metric Reference

The JGA therefore models musical analysis probabilistically rather than deterministically, explicitly representing the uncertainty naturally present in real musical performances.
