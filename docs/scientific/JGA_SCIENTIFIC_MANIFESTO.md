# JGA Scientific Manifesto

Status: Draft

## Purpose

This document defines the scientific philosophy of the Jazz Groove Analyzer (JGA).

It establishes the principles that guide the scientific model independently from the software implementation.

The Manifesto explains *why* the JGA has been designed, while the Observation Model explains *how* musical performances are scientifically observed.

---

## Vision

The Jazz Groove Analyzer aims to provide a scientific framework for the analysis of rhythmic behaviour in jazz ensembles.

The objective is not to detect beats or classify musical styles, but to describe how metric organization emerges from observable musical evidence.

The JGA treats musical performance as an observable phenomenon that can be studied through reproducible scientific methods.

---

## Scientific Philosophy

The JGA separates observation from interpretation.

Observable evidence always precedes musical meaning.

Every semantic conclusion must be supported by observable temporal evidence.

The scientific model is independent from any particular DSP algorithm or software implementation.

---

## Fundamental Principles

- Observation before interpretation.
- Theory precedes implementation.
- Musical meaning emerges from observable evidence.
- Scientific models must be reproducible.
- Every analytical step must be explicit.
- Scientific concepts must remain independent from implementation details.

---

## Observation Before Interpretation

The JGA never assigns musical meaning directly from the audio signal.

Observation produces objective evidence.

Interpretation is performed only after sufficient observable evidence has been established.

---

## Theory Before Implementation

The scientific model defines the software architecture.

Implementation never defines the scientific theory.

Every architectural decision must be consistent with the scientific model.

---

## Scientific Consequences

The separation between observation, computation and semantic interpretation is mandatory.

The scientific model remains stable even when algorithms evolve.

Computational improvements must not modify the scientific meaning of the analytical model.

---

## Relationship with the Observation Model

The Observation Model defines how observable evidence is constructed.

This Manifesto defines the scientific principles that justify that process.

---

## Relationship with the Domain Model

The Domain Model represents the semantic interpretation of observable evidence.

It never replaces scientific observation.

---

## Relationship with the Software Architecture

The software architecture exists to implement the scientific model.

Architectural decisions must always preserve the separation between observation, translation and domain semantics.
