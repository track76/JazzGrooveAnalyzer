# Scientific Visualization

Status
    PROPOSED

Version
    1.0

Author
    Angelo Tracanna

------------------------------------------------------------
Purpose
------------------------------------------------------------

The Jazz Groove Analyzer does not generate visualizations
for aesthetic purposes.

Every graphical element must represent an observable
scientific entity belonging to the JGA theoretical model.

The visualization layer is therefore considered part of the
scientific architecture.

------------------------------------------------------------
Fundamental Principles
------------------------------------------------------------

1.
Every visualization must be generated exclusively from
objects belonging to the scientific model.

2.
No graphical element may introduce information that is not
contained in the observable data.

3.
Every graphical element must be traceable back to the
Elementary Metric Events that generated it.

4.
The visualization layer never performs musical
interpretation.

It only represents observable phenomena.

------------------------------------------------------------
Metric Cluster Portrait
------------------------------------------------------------

Each Metric Cluster shall be represented as a scientific
portrait.

The Beat Reference constitutes the geometrical reference of
the portrait.

The Beat Reference shall be represented by a vertical line.

Every Elementary Metric Event shall be represented as a
point positioned according to its temporal distance from the
Beat Reference.

For every Elementary Metric Event the visualization shall
preserve:

- source identity
- instrument identity
- drum component identity
- musical function
- metric contribution
- temporal offset expressed in milliseconds

------------------------------------------------------------
Metric Landscape
------------------------------------------------------------

A musical performance shall be represented as a sequence of
Metric Cluster Portraits.

This representation is called the Metric Landscape.

The Metric Landscape allows observation of:

- evolution of the internal tempo

- evolution of Metric Clusters

- evolution of Elementary Metric Events

- evolution of instrument behaviour

- evolution of rhythmic roles

------------------------------------------------------------
Behaviour Anomaly Detection
------------------------------------------------------------

The visualization layer shall support automatic detection of
behavioural anomalies.

An anomaly is NOT considered an execution error.

An anomaly is defined as an Elementary Metric Event or a
Metric Cluster whose behaviour significantly differs from
the statistical behaviour observed during the complete
performance.

Possible anomaly categories include:

- temporal anticipation

- temporal delay

- abnormal cluster width

- abnormal cluster density

- source behaviour changes

- rhythmic regime transitions

------------------------------------------------------------
Scientific Report
------------------------------------------------------------

The visualization layer shall support generation of a
complete scientific report.

The report shall include:

- global statistics

- internal tempo evolution

- Metric Landscape

- instrument analysis

- drum component analysis

- anomaly analysis

- Behaviour Analytics summary

------------------------------------------------------------
Natural Language
------------------------------------------------------------

Natural language descriptions shall always be generated
AFTER scientific analysis.

Natural language never replaces scientific measurements.

Every sentence must remain fully traceable to the observable
data that generated it.


------------------------------------------------------------
Future Scientific Capabilities
------------------------------------------------------------

The Scientific Visualization Layer shall progressively
support the following scientific representations.

Metric Cluster Portrait

Every Metric Cluster shall be represented as a scientific
portrait centered on the Beat Reference.

The Beat Reference shall be represented as a vertical line.

Every Elementary Metric Event shall preserve its identity
and shall be represented according to its temporal offset
expressed in milliseconds.

Metric Landscape

The complete musical performance shall be represented as a
continuous sequence of Metric Cluster Portraits.

The Metric Landscape shall allow visual inspection of the
evolution of rhythmic behaviour throughout the performance.

Instrument Behaviour

The visualization layer shall preserve the behaviour of
every individual instrument.

Drum components shall be represented independently whenever
they contribute to metric construction.

Behaviour Anomaly Detection

The system shall automatically identify statistically
significant Elementary Metric Events and Metric Clusters.

Scientific Report

The visualization layer shall generate a complete scientific
report containing measurements, visualizations and
interpretation.

Natural Language

Natural language descriptions shall always be derived from
scientifically validated analytical results.

Every sentence shall remain traceable to the Elementary
Metric Events that generated it.
